# Техническое задание — Hermes Desktop

> **Статус:** Черновик — будет заполнен через spec-driven-development

## Цель

Нативный macOS-клиент для удалённого Hermes Agent (VPS), который решает проблемы Telegram-интерфейса и добавляет мультиагентскую оркестрацию.

## Пользовательские сценарии

### v1.0 (MVP)

1. **Подключение к API** — пользователь вводит URL и API-ключ, приложение коннектится к Hermes на VPS
2. **Чат со стримингом** — отправка сообщений, streaming ответов (SSE), Markdown-рендеринг
3. **Топики** — создание/удаление топиков, каждый топик = отдельная сессия
4. **История** — просмотр прошлых сессий с поиском
5. **Git-синк личности** — при запуске проверяет agents-hub, подтягивает обновления памяти/скиллов

### v1.1 (Multi-agent)

6. **Быстрые действия** — кнопки «Review», «Implement», «Test»
7. **Мониторинг subagent'ов** — панель со статусом запущенных агентов
8. **Результаты ревью** — inline-отображение результатов code review

## Не-функциональные требования

- macOS 14 Sonoma+
- Apple Silicon native
- API-ключ хранится в macOS Keychain
- Офлайн-доступ к топикам и истории (SwiftData)
- Минимум зависимостей

## Границы (что НЕ делаем)

- НЕ встраиваем свой LLM-движок — всё через API
- НЕ поддерживаем iOS/iPadOS в v1
- НЕ делаем веб-версию
- НЕ заменяем Hermes CLI — дополняем
