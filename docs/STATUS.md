# Hermes Desktop — Project Status

> **Дата:** 2026-07-11  
> **Сессия:** 2026-07-11

---

## 1. Сделано сегодня

- Единая модель `Chat` с закреплением — Темы и Чаты объединены в одну
  сущность (`isPinned`), в сайдбаре секция «Закреплённые» поверх
  обычных чатов.
- Sessions API для новых чатов — обычные (незакреплённые) чаты идут
  через новый Sessions API, закреплённые продолжают работать через
  старый Runs API.
- Английский интерфейс — весь текст, видимый на экране, переведён на
  английский; документация и общение с владельцем — по-прежнему на
  русском (закреплено постоянным правилом в CLAUDE.md).
- Фикс SSE-парсера (`SSEClient`) — ушли от `bytes.lines` (на этой
  платформе не отдавал пустые строки — разделитель события по спеке
  SSE), парсинг теперь построчный вручную, с накоплением нескольких
  `data:`-строк на одно событие.
- Троттлинг отрисовки стриминга — `ChatViewModel` публикует
  `streamingContent` не чаще раза в ~70мс вместо каждой дельты;
  автоскролл развязан от этого троттлинга отдельным счётчиком.

## 2. Итог по стримингу

**В беклоге, не решено.** Владелец проверил вживую: текст всё ещё
появляется одним блоком, а не построчно. При этом сам стриминг данных
подтверждён логом реального запроса — 676 обновлений
`streamingContent` равномерно за 8.4 секунды, без пропусков. Значит,
проблема не в сети и не в парсинге SSE, а где-то в отрисовке
(`ChatView`/`StreamingText`/`MarkdownRenderer`) — подробности и
гипотезы см. `docs/UI-SPEC.md` §10 п.6.

## 3. Следующий большой этап — Проекты (Этап 3)

Группа чатов с общими инструкциями (system prompt поверх личности
агента) и общей памятью между чатами внутри проекта
(`X-Hermes-Session-Key`). Техническая разведка по API Hermes завершена,
механизмы подтверждены — можно переходить к реализации.

## 4. Технический долг

- `SessionSSEClient` (Sessions API) почти наверняка страдает тем же
  SSE-багом, что чинили в `SSEClient` — тот же паттерн `bytes.lines`.
  В этот раз лечить общим переиспользуемым компонентом, а не
  дублировать фикс во второй раз.
- Миграция закреплённых чатов со старого Runs API на Sessions API —
  см. `docs/UI-SPEC.md` §10 п.3.
- Автоскролл во время стриминга не учитывает ручную прокрутку вверх —
  см. `docs/UI-SPEC.md` §10 п.5.
