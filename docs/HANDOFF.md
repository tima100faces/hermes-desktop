В ветке `feature/project-management` (пока не в main, ждёт подтверждения
билда владельцем перед мержем):
- Починен баг: `lastActiveAt` раньше выставлялся только при создании
  проекта и больше никогда не обновлялся — «последний активный» на деле
  был «последний созданный». Теперь обновляется при каждой отправке и
  получении сообщения (ChatViewModel).
- Управление проектами: иконка «…» в строке проекта (по наведению) +
  правый клик — меню Rename / Delete. Rename — модальное окно
  (RenameProjectSheet), Delete — как раньше, с подтверждением. Владелец
  проверил вживую — работает.
- Архивирование обсудили и осознанно решили не делать (см. UI-SPEC §10,
  «Решено и закрыто»).
- Починен баг автоскролла: при отправке сообщения скролл иногда
  оставался на последнем сообщении, а точка «думает» уезжала ниже
  экрана (гонка между двумя `onChange`). Теперь автоскролл при отправке
  всегда целится в индикатор стриминга, если run активен.
- Добавлена кнопка «прокрутить вниз» (`ScrollToBottomButton`, ChatView.swift):
  появляется, когда чат прокручен вверх, стоит над кнопкой отправки.
  Владелец проверил вживую — работает.
- Добавлена палитра ⌘K (`ProjectPaletteView`) — быстрый переход между
  проектами с фильтром по вводу. По ходу всплыл и починен баг: активный
  проект дублировался в двух местах состояния (SidebarViewModel и
  ContentView), из-за чего выбор через палитру не подсвечивался в
  сайдбаре. Теперь `selectedProject` — общий `@Binding`, один источник
  истины. Владелец проверил вживую — работает.
- Shift+Enter теперь вставляет перенос строки по позиции курсора:
  поле ввода переведено на `GrowingTextEditor` (обёртка над NSTextView,
  Features/Chat/GrowingTextEditor.swift) вместо SwiftUI TextField.
  Заодно по просьбе владельца поле растёт не до фиксированных 8 строк,
  а до половины высоты панели чата (как в приложении Клода) — дальше
  скроллится внутри себя. Владелец проверил вживую — работает.

## Правила (коротко; полностью — CLAUDE.md и docs/UI-SPEC.md)

- Перед любым UI-кодом — прочитать UI-SPEC.md, в конце — чек-лист.
- Работа в feature-ветках; в main — только после подтверждения владельца.
- После изменений — `make build`; если менялся UI — обновить спеку.
- Решения из «Решено и закрыто» (UI-SPEC §10) не пересматривать.
- Владелец — дизайнер, не программист: объяснять просто, по-русски,
  визуальные изменения описывать до реализации.
