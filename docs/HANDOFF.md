## Беклог — по приоритету

Брать по одной задаче за раз, каждая — в своей feature-ветке.

### 1. Retry в hover-ряду
Иконка `arrow.clockwise` (IconActionButton) в hover-ряду ПОСЛЕДНЕГО ответа
ассистента. Клик → повторная отправка последнего СООБЩЕНИЯ ПОЛЬЗОВАТЕЛЯ
(нужен метод в ChatViewModel; старый ответ НЕ удалять, новый приходит
следующим). Стиль кнопки — как у Copy (UI-SPEC §5).

### 2. Cmd+K — палитра проектов
Оверлей по центру окна: поле поиска + список проектов, фильтр по вводу,
стрелки + Enter для выбора, Esc — закрыть. Стиль: карточка `hkSurface`,
радиус 12, `hkGlowStrong`, ширина ~480, затемнение фона black 40%.
Выбранный элемент — подложка `hkAccentDim` (как в сайдбаре).
Это единственное допустимое исключение из правила «только inset glow»:
оверлею можно мягкую тень `hkPage` для отделения от фона — но сначала
показать владельцу вариант без тени.

### 3. Shift+Enter по позиции курсора
Сейчас перенос вставляется в конец текста (ограничение SwiftUI
TextField). Решение — NSTextView-обёртка (NSViewRepresentable). Объёмная
задача, не совмещать с другими.

### 4. Подсветка синтаксиса в CodeBlockView
Минималистичная: ключевые слова — `hkAccent2`, строки — `hkSuccess`,
комментарии — `hkNeutral`, остальное — `hkMuted`. Без тяжёлых зависимостей;
если без библиотеки никак — сначала обсудить с владельцем.

## Правила (коротко; полностью — CLAUDE.md и docs/UI-SPEC.md)

- Перед любым UI-кодом — прочитать UI-SPEC.md, в конце — чек-лист.
- Работа в feature-ветках; в main — только после подтверждения владельца.
- После изменений — `make build`; если менялся UI — обновить спеку.
- Решения из «Решено и закрыто» (UI-SPEC §10) не пересматривать.
- Владелец — дизайнер, не программист: объяснять просто, по-русски,
  визуальные изменения описывать до реализации.
