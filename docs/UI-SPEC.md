# Hermes Desktop — UI Spec (Rust / Obsidian-derived)

> Спецификация интерфейса. Обязательна к прочтению для любого агента,
> который меняет UI-код. Отклонения от спеки = баг.

## 1. Философия

Дизайн основан на стиле Obsidian (Refero: «Crystalline Knowledge Vault»)
с собственным акцентным цветом — **ржавым** (Rust), в честь агента
Ржавчика. Тёмный, контрастный, утилитарный интерфейс инструмента, а не
маркетингового сайта.

Ключевые принципы:

1. **Никаких хардкодов.** Все цвета — только токены из
   `DesignSystem/Colors.swift` (`Color.hk*`), все отступы — `Space.*`,
   все шрифты — `Font.hk*`.
2. **Никакой цветовой математики в рантайме.** Токены задаются только
   литеральным hex через `Color(hk:)`. Ранее в проекте была
   oklch→sRGB конвертация с багом (пропущено кубирование LMS и вторая
   матрица), из-за которого весь интерфейс рендерился в ~3 раза светлее.
   Не возвращать.
3. **Elevation через внутреннее свечение, не тени.** Карточки и кнопки
   получают 1px inset glow (`hkGlow` / `hkGlowStrong`), drop shadows
   запрещены.
4. **Акцент — только на интерактиве и живых процессах.** Кнопка
   отправки, выделенный проект, бейдж subagent'ов, спиннеры, ссылки.
   Никогда — на фонах сообщений или декоративных элементах.
5. **Системный шрифт** (SF Pro / SF Mono). Никаких кастомных шрифтов.
6. **Радиусы:** карточки 12, инпуты/кнопки/код 8, бейджи — капсула.
   Больше 12 — нельзя.
7. **Hover-зоны:** любой контейнер с `.onHover` обязан иметь
   `.contentShape(Rectangle())`, иначе прозрачные зоны теряют курсор и
   hover-элементы исчезают до клика.

## 2. Цвета (литеральный hex, sRGB)

### Нейтральные

| Токен | Hex | Роль |
|---|---|---|
| `hkPage` | #171717 | Фон окна |
| `hkPanel` | #1a1a1a | Сайдбар |
| `hkSurface` | #1e1e1e | Карточки: сообщение пользователя, инпут, статус subagent'а |
| `hkSurface2` | #262626 | Приподнятые поверхности, disabled-кнопка отправки |
| `hkCodeBg` | #121212 | Фон блоков кода — темнее страницы |
| `hkBorder` | #2e2e2e | Границы |
| `hkRule` | #3f3f3f | Видимые разделители |
| `hkInk` | #eeeeee | Основной текст |
| `hkMuted` | #bcbcbc | Вторичный текст |
| `hkNeutral` | #a3a3a3 | Третичный текст: таймстампы, плейсхолдеры, метки |

### Акцент — Rust (пара, как Amethyst/Lavender у Obsidian)

| Токен | Hex | Роль |
|---|---|---|
| `hkAccent` | #b7410e | **Только заливки**: кнопка отправки, полоска выделенного проекта |
| `hkAccent2` | #d97e52 | **Только текст/иконки**: ссылки, активные подписи, спиннеры, ключевые слова в коде |
| `hkAccentDim` | rust 16% | Подложки: выделение в сайдбаре, бейджи, кнопка Stop |

⚠️ Тёмный rust (#b7410e) нечитаем как мелкий текст на тёмном фоне —
для текста всегда `hkAccent2`.

### Семантические

| Токен | Hex | Роль |
|---|---|---|
| `hkSuccess` | #4ade80 | Точка online, завершённые subagent'ы |
| `hkWarning` | #facc15 | Только текст, никогда заливка (конфликтует с rust) |
| `hkError` | #f87171 | Ошибки |

### Эффекты

| Токен | Роль |
|---|---|
| `hkGlow` (white 5%) | Inset glow карточек |
| `hkGlowStrong` (white 8%) | Inset glow инпута |
| `hkHover` (white 4%) | Hover-подложка ghost-кнопок |

## 3. Анатомия сообщений

| Роль | Вид |
|---|---|
| Пользователь | Карточка `hkSurface`, радиус 12, inset glow, прижата вправо, max 580pt, текст `hkInk` |
| Ассистент | **Без пузыря.** Плоский markdown (`MarkdownRenderer`) на фоне страницы, прижат влево, max 580pt |
| Tool | Курсив по центру, `hkNeutral`, caption |

- Весь текст selectable (`.textSelection(.enabled)`).
- Таймстампы НЕ показываются под каждым сообщением — только в
  hover-ряду действий.

## 4. Блоки кода (`CodeBlockView`)

- Фон `hkCodeBg` (темнее страницы), радиус 8, inset glow.
- Шапка: язык слева (`hkCodeCaption`, `hkNeutral`), иконка Copy справа.
- Тело: SF Mono (`hkCodeBody`), цвет `hkMuted`, горизонтальный скролл.
- Копирование по клику: иконка меняется на checkmark на 1.5с.

## 5. Hover-ряд действий

Под каждым сообщением (кроме tool) при наведении появляется ряд:
- Иконка Copy (26×26, ghost, `hkHover` при наведении) — **без текстовых
  подписей, только иконки**
- Таймстамп (`hkCaption`, `hkNeutral`)
- Появление/исчезание: opacity, 0.12s easeOut
- Зона наведения — всё сообщение + ряд + зазоры, через
  `.contentShape(Rectangle())` на внешнем контейнере (см. §1 п.7)

Будущие действия (Retry и т.п.) добавляются в этот же ряд как
`IconActionButton`.

## 6. Поле ввода (`InputBar`)

- Одна карточка `hkSurface`, радиус 8, `hkGlowStrong`, лежит на `hkPage`
  без панели-подложки и без верхнего разделителя.
- TextField растёт от 1 до 8 строк.
- **Enter — отправить, Shift+Enter — новая строка** (через
  `.onKeyPress(.return)`; известное ограничение: Shift+Enter вставляет
  перенос в конец текста, не по позиции курсора).
- Кнопка отправки 28×28: `hkAccent` + белая стрелка; пустой инпут —
  `hkSurface2` + disabled; при стриминге — Stop (`hkAccentDim` +
  `hkAccent2`).

## 7. Subagent'ы

Статус — карточка `hkSurface` в потоке чата (радиус 12, inset glow):
- running → спиннер `hkAccent2` + имя (`hkInk`, medium) + прогресс
  (`hkCaption`, `hkNeutral`, 1 строка)
- завершён → `checkmark.circle` цвета `hkSuccess`

В шапке чата — капсульный бейдж «N subagents» (`hkAccentDim` фон,
`hkAccent2` текст), виден только когда N > 0.

## 8. Каркас окна

- `.windowStyle(.hiddenTitleBar)` + `.preferredColorScheme(.dark)`.
- Раскладка — **обычный HStack, НЕ NavigationSplitView**: системный
  сплит-вью накладывает материалы macOS поверх `hkPanel` (именно из-за
  этого старый билд выглядел серым). Не возвращать NavigationSplitView.
- Сайдбар: фикс. ширина 220, фон `hkPanel`, сверху 36pt под
  traffic lights.
- Разделитель сайдбар/чат: 1px white 6%.
- Заголовок проекта — в шапке чата, не в системном тайтлбаре.

## 9. Сайдбар

- Список проектов — **ScrollView + кнопки, НЕ List**: нативный List
  рисует своё системное выделение поверх кастомного. Не возвращать.
- Шапка: «PROJECTS» (11pt semibold, uppercase, tracking 0.8,
  `hkNeutral`) + кнопка «+».
- Строка проекта: имя (`hkBody` medium) + относительное время
  (`hkCaption`).
- Выделение: скруглённая подложка `hkAccentDim` (радиус 8) + rust-полоска
  2pt слева + время цветом `hkAccent2` + имя `hkInk`.
- Невыделенные: имя `hkMuted`, время `hkNeutral`, без фона.
- Скроллбары скрыты (`.scrollIndicators(.hidden)`) — и здесь, и в чате.
- Футер: точка статуса + «Ржавчик» (сейчас статично; см. §10).

## 10. Следующие этапы (не реализовано, не удалять из спеки)

1. Привязать футер сайдбара к реальному health-check Hermes API
   (зелёный = online, `hkNeutral` = offline) и имени агента из
   личности (agents-hub).
2. Кнопка Retry в hover-ряду (повторная отправка последнего запроса).
3. Cmd+K — палитра быстрого перехода по проектам.
4. Empty state нового проекта с быстрыми действиями (ревью,
   имплементация).
5. Shift+Enter с учётом позиции курсора (NSTextView-обёртка).
6. Подсветка синтаксиса в блоках кода (ключевые слова — `hkAccent2`).

## Чек-лист для агента перед коммитом UI-кода

- [ ] Ни одного литерального цвета/hex в компонентах — только `Color.hk*`
- [ ] Никакой цветовой математики (oklch и пр.) — только литеральный hex в Colors.swift
- [ ] Ни одного магического отступа — только `Space.*` (±2pt допустимо)
- [ ] Ни одной drop shadow — только inset glow
- [ ] Радиусы: 12 карточки, 8 инпуты/кнопки/код, капсула — бейджи
- [ ] Текст сообщений selectable
- [ ] Акцентный текст — `hkAccent2`, акцентные заливки — `hkAccent`
- [ ] У каждого `.onHover`-контейнера есть `.contentShape(Rectangle())`
