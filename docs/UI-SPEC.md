# Hermes Desktop — UI Spec (Rust / Obsidian-derived)

> Спецификация интерфейса. Обязательна к прочтению для любого агента,
> который меняет UI-код. Отклонения от спеки = баг.

## 1. Философия

Дизайн основан на стиле Obsidian (Refero: «Crystalline Knowledge Vault»)
с собственным акцентным цветом — **ржавым** (Rust), в честь агента
Ржавчика. Тёмный, контрастный, утилитарный интерфейс инструмента, а не
маркетингового сайта.

Ключевые принципы:

1. **Никаких хардкодов.** Все цвета — только токены из
   `DesignSystem/Colors.swift` (`Color.hk*`), все отступы — `Space.*`,
   все шрифты — `Font.hk*`.
2. **Никакой цветовой математики в рантайме.** Токены задаются только
   литеральным hex через `Color(hk:)`. Ранее в приложении была
   oklch→sRGB конвертация с багом (пропущено кубирование LMS и вторая
   матрица), из-за которого весь интерфейс рендерился в ~3 раза светлее.
   Не возвращать.
3. **Elevation через внутреннее свечение, не тени.** Карточки и кнопки
   получают 1px inset glow (`hkGlow` / `hkGlowStrong`), drop shadows
   запрещены.
4. **Акцент — только на интерактиве и живых процессах.** Кнопка
   отправки, выделенная тема, бейдж subagent'ов, спиннеры, ссылки,
   индикатор размышления. Никогда — на фонах сообщений или
   декоративных элементах.
5. **Системный шрифт** (SF Pro / SF Mono). Никаких кастомных шрифтов.
6. **Радиусы:** карточки 12, инпуты/кнопки/код 8, бейджи — капсула.
   Больше 12 — нельзя.
7. **Hover-зоны:** любой контейнер с `.onHover` обязан иметь
   `.contentShape(Rectangle())`, иначе прозрачные зоны теряют курсор и
   hover-элементы исчезают до клика.
8. **`repeatForever`-анимации** запускать через `withAnimation` внутри
   `.task` — паттерн `onAppear` + `.animation(value:)` молча не стартует.

## 3. Анатомия сообщений

| Роль | Вид |
|---|---|
| Пользователь | Карточка `hkSurface`, радиус 12, inset glow, прижата вправо, max 580pt, текст `hkInk` |
| Ассистент | **Без пузыря.** Плоский markdown (`MarkdownRenderer`) на фоне страницы, прижат влево, max 580pt |
| Tool | Курсив по центру, `hkNeutral`, caption |

- Весь текст selectable (`.textSelection(.enabled)`).
- Таймстампы НЕ показываются под каждым сообщением — только в
  hover-ряду действий.
- **Пустая тема = чистый чат.** Никаких placeholder'ов, быстрых
  действий и подсказок — осознанное решение владельца. Не добавлять.
- **Индикатор размышления** (`ThinkingIndicator`): пока run активен, но
  токены ещё не пришли — пульсирующая точка `hkAccent2` (10pt,
  scale 0.75↔1.25, opacity 1↔0.45, easeInOut 0.8s autoreverse) слева в
  потоке чата. Анимация стартует через `withAnimation` в `.task`
  (см. §1 п.8). Как только пошёл первый токен — сменяется стриминговым
  текстом.
  - При отправке сообщения одновременно меняются `messages.count` и
    `isStreaming` — автоскролл должен целиться в `"streaming"`, а не в
    последнее сообщение, иначе точка остаётся ниже видимой области.
    См. `.onChange(of: viewModel.messages.count)` в ChatView.swift.
- **Кнопка «прокрутить вниз»** (`ScrollToBottomButton`): круглая 32×32,
  `hkSurface` + `hkGlowStrong`
  обводка, иконка `chevron.down` (не `arrow.down` — со стрелкой
  отправки в InputBar смотрится как парный тумблер). Показывается,
  только когда чат прокручен вверх (невидимый маркер `"bottom"` в
  конце списка через onAppear/onDisappear). Расположение — по центру
  над кнопкой отправки (trailing-отступ = отступы InputBar минус
  половина разницы в размере кнопок), на той же высоте, что и низ
  списка сообщений. Появление/исчезание — opacity, easeOut 0.12s.
  `.contentShape(Circle())` обязателен — без него кликабельна только
  сама иконка стрелки, а не весь круг (нашли и починили 2026-07-12).

## 4. Блоки кода (`CodeBlockView`)

- Фон `hkCodeBg` (темнее страницы), радиус 8, inset glow.
- Шапка: язык слева (`hkCodeCaption`, `hkNeutral`), иконка Copy справа.
- Тело: SF Mono (`hkCodeBody`), горизонтальный скролл.
- Копирование по клику: иконка меняется на checkmark на 1.5с.
- **Подсветка синтаксиса** (`SyntaxHighlighter`): ключевые слова —
  `hkAccent2`, строки — `hkSuccess`, комментарии — `hkNeutral`,
  остальное — `hkMuted`. Самодельный однопроходный токенайзер (без
  внешних библиотек) — распознаёт комментарии/строки и красит слова
  по одному общему списку ключевых слов на все языки сразу (не
  парсер по грамматике конкретного языка), так что изредка подсвечивает
  чужое ключевое слово не в тему — осознанный компромисс за простоту.

## 5. Hover-ряд действий

Под каждым сообщением (кроме tool) при наведении появляется ряд:
- Иконка Copy (26×26, ghost, `hkHover` при наведении) — **без текстовых
  подписей, только иконки**
- Таймстамп (`hkCaption`, `hkNeutral`)
- Появление/исчезание: opacity, 0.12s easeOut
- Зона наведения — всё сообщение + ряд + зазоры, через
  `.contentShape(Rectangle())` на внешнем контейнере (см. §1 п.7)

Будущие действия (Retry и т.п.) добавляются в этот же ряд как
`IconActionButton`.

## 6. Поле ввода (`InputBar`)

- Одна карточка `hkSurface`, радиус 8, `hkGlowStrong`, лежит на `hkPage`
  без панели-подложки и без верхнего разделителя.
- Поле — `GrowingTextEditor` (обёртка над `NSTextView`, не SwiftUI
  `TextField`): растёт от 1 строки до **половины высоты панели чата**
  (как в приложении Клода), дальше — прокрутка внутри поля. Высота
  панели приходит из `GeometryReader` в `ChatView` и пересчитывается
  при ресайзе окна.
- **Enter — отправить, Shift+Enter — перенос строки по месту курсора**
  (перехват на уровне AppKit в `SendAwareTextView.keyDown`, не
  `.onKeyPress` — так перенос вставляется именно там, где курсор, а не
  в конец текста).
- Placeholder «Message» — обычный SwiftUI `Text` поверх редактора
  (`hkNeutral`), не встроен в NSTextView.
- Кнопка отправки 28×28: `hkAccent` + белая стрелка; пустой инпут —
  `hkSurface2` + disabled; при стриминге — Stop (`hkAccentDim` +
  `hkAccent2`).
- `GrowingTextEditor` также переиспользуется полем инструкций на
  странице проекта (§9 «Проекты») — единственный способ точно измерить
  растущий текст. Расширен двумя необязательными параметрами с
  дефолтами, полностью сохраняющими это, InputBar'овское, поведение:
  `autoFocus` (по умолчанию `true`) и `onPlainReturn` (по умолчанию
  `nil` — если не задан, Enter вставляет перенос строки как обычно,
  вместо отправки).

## 7. Subagent'ы

Статус — карточка `hkSurface` в потоке чата (радиус 12, inset glow):
- running → спиннер `hkAccent2` + имя (`hkInk`, medium) + прогресс
  (`hkCaption`, `hkNeutral`, 1 строка)
- завершён → `checkmark.circle` цвета `hkSuccess`

В хедере контент-панели (`ContentHeaderView`, см. §9 «Единый хедер») —
капсульный бейдж «N subagents» в правом краю (`hkAccentDim` фон,
`hkAccent2` текст), виден только когда N > 0, у обоих чат-состояний
(с проектом и без); на странице проекта его нет. Название/крошка
уступают ему место при усечении, сам бейдж не уменьшается.

## 8. Каркас окна

- `.windowStyle(.hiddenTitleBar)` + `.preferredColorScheme(.dark)`.
- Раскладка — **обычный HStack, НЕ NavigationSplitView**: системный
  сплит-вью накладывает материалы macOS поверх `hkPanel` (именно из-за
  этого старый билд выглядел серым). Не возвращать NavigationSplitView.
- Сайдбар: фикс. ширина 220, фон `hkPanel`, сверху 36pt под
  traffic lights.
- Разделитель сайдбар/чат: 1px white 6%.
- Заголовок темы — в шапке чата, не в системном тайтлбаре.

## 9. Сайдбар

Одна сущность — **Чат** (`Chat`, SwiftData). Две секции сверху вниз:
закреплённые чаты (заголовок «Pinned», только когда есть хотя бы один
закреплённый чат) и обычные чаты (заголовок «Chats») — обе через один
и тот же анатомический ряд (`ConversationRowContent`, общий для всех
строк `ChatRow`), чтобы
визуальная правка не делалась дважды. Оба списка — **ScrollView +
кнопки, НЕ List**: нативный List рисует своё системное выделение поверх
кастомного. Не возвращать.

- У каждого чата один из двух транспортов, определяется тем, какое поле
  установлено: `conversationKey` — старый путь (Runs API, остался от
  прежней отдельной сущности «Тема»), `sessionId` — новый путь (Sessions
  API). Этот выбор скрыт за `ConversationService` — снаружи разницы нет.
- **Закреплённые чаты** — чаты с `isPinned == true`, всегда выше
  обычных, без своей кнопки «+» (закрепление — только через меню
  существующего чата). Все чаты, смигрированные из старой сущности
  «Тема», закреплены по умолчанию и продолжают работать через старый
  Runs API — перенос их на Sessions API не входил в эту задачу (см.
  §10).
- **Обычные чаты** — все остальные (незакреплённые) чаты, всегда на
  новом пути (Sessions API). Список в сайдбаре — **всегда локальный** (записи
  `Chat` в SwiftData), никогда не строится через `GET /api/sessions`:
  этот эндпоинт также возвращает сессии из Telegram, а для чатов на
  api_server оба типа помечены одинаковым `source` — надёжно отличить
  их друг от друга по ответу сервера нельзя.
- **При запуске автоматически выбирается самая недавняя активная
  беседа** — по `lastActiveAt`, вне зависимости от секции — приложение
  открывается сразу в чате, а не в заглушке. Заглушка («Start your
  first chat») остаётся только когда чатов нет вовсе; если чаты есть,
  но выбор снят (например, удалили открытый чат, а другие остались) —
  другая заглушка («Select a chat»).
- Шапка секции: название (рендерится капсом через `textCase`, 11pt
  semibold, uppercase, tracking 0.8, `hkNeutral`). У «Chats» — кнопка
  «+»: сразу создаёт чат (`POST /api/sessions`) и открывает его — без
  модального окна, заголовок появится сам после первого сообщения
  (автозаголовок; дефолтный заголовок нового чата — «New chat»,
  `Chat.defaultTitle`). Кнопка «+» в сайдбаре одна; у «Pinned» своей
  кнопки нет.
  - **Автозаголовок** (`SessionsConversationService.autoTitleIfNeeded`)
    — сервер сам заголовок не генерирует (`title` в `GET
    /api/sessions/{id}` остаётся `null` даже после `run.completed`,
    проверено вживую 2026-07-12), поэтому заголовком становится текст
    первого сообщения: пробелы/переносы схлопываются в один пробел,
    результат обрезается до 40 символов по границе слова (не
    посередине) с «…» в конце; если пробела в пределах 40 символов нет
    вовсе — жёсткая обрезка. Ставится сразу после отправки первого
    сообщения (`PATCH /api/sessions/{id}` + локально), не дожидаясь
    ответа агента. Срабатывает, только если заголовок чата всё ещё
    буквально «New chat» — проверяется прямо перед записью (а не
    только через флаг `Chat.hasAutoTitled`), чтобы ручное
    переименование, случившееся, пока запрос был в полёте, никогда не
    затёрлось. Применяется и к чатам проектов — у них тот же
    транспорт.
- Строка беседы: имя/заголовок (`hkBody` medium) + относительное время
  (`hkCaption`).
- Выделение: скруглённая подложка `hkAccentDim` (радиус 8) + rust-полоска
  2pt слева + время цветом `hkAccent2` + имя `hkInk`.
- Невыделенные: имя `hkMuted`, время `hkNeutral`, без фона.
- Скроллбары скрыты (`.scrollIndicators(.hidden)`) — и здесь, и в чате.
- **Футер**: точка статуса (`hkSuccess` online / `hkError` offline /
  `hkNeutral` ещё не проверено; источник — `ConnectionMonitor`,
  `GET /v1/health` раз в 30с) + имя агента из настроек
  (`@AppStorage("agent_name")`, без хардкода по умолчанию — пустое
  значение отображается как «Hermes», реальное имя задаётся в
  Settings). Высота футера — константа, ничего в нём её не меняет.
  - **Кнопка настроек** — иконка `gearshape` (13pt, `hkMuted`) справа,
    отступ от края панели — `Space.sm` (тот же, что у контента
    сайдбара, а не `Space.md`, которым инсетится сама строка футера).
    Hover — цвет `hkInk`, фон `hkSurface`, скругление 6pt, padding
    4pt. Клик — `@Environment(\.openSettings)` (то же самое, что
    Cmd+,). Тултип «Settings». Реализована как `.overlay(alignment:
    .trailing)` поверх строки футера, а не как элемент внутри её
    HStack — свой хит-бокс кнопки (13+4+4=21pt) выше нынешней высоты
    строки, и только overlay не участвует в расчёте высоты родителя;
    строка резервирует под кнопку 21+8=29pt справа, чтобы длинное имя
    агента не наезжало на неё визуально.
- **Управление чатом**: при наведении на строку справа появляется
  иконка «…» (`ConversationMenuButton`, тот же ghost-стиль 26×26, что и
  `IconActionButton` в hover-ряду сообщений) — открывает меню «Rename» /
  «Pin»-«Unpin» / «Delete». То же самое доступно через правый клик
  (`contextMenu`).
  - «Rename» — `RenameChatSheet` (заголовок «Rename chat», поле
    «Chat name»). Для чата на Sessions API уходит на сервер (`PATCH
    /api/sessions/{id}`) и одновременно отключает автозаголовок
    (`Chat.hasAutoTitled`); для закреплённого чата на Runs API — только
    локально (как раньше у Тем).
  - «Pin»/«Unpin» — переключает `Chat.isPinned`, всегда локально, без
    обращения к серверу, для обоих транспортов.
  - «Delete» — алерт подтверждения «Delete chat?» («This chat will be
    permanently deleted.»). Для чата на Sessions API уходит на сервер
    (`DELETE /api/sessions/{id}`); для закреплённого чата на Runs API —
    только локально.
  - `lastActiveAt` обновляется при каждой отправке/получении сообщения
    — не только при создании чата.
- **Активный чат — общее состояние (`@Binding`), не дублируется.**
  `SidebarView` получает `selection: Chat?` биндингом из `ContentView`,
  а не хранит свой собственный источник истины — иначе способы выбора в
  обход сайдбара (например ⌘K-палитра) не обновляют подсветку в
  списке. Не возвращать по отдельному состоянию выбора в
  `ChatSidebarViewModel` — уже наступали на эти грабли.
- **Cmd+K — палитра быстрого перехода** (`ChatPaletteView`) — ищет по
  **всем** чатам, закреплённым и обычным (до объединения искала только
  по темам — с одним списком вместо двух отдельного смысла сужать её
  уже не было). Оверлей по центру окна, затемнение фона black 40%
  (клик — закрыть). Карточка `hkSurface`, радиус 12, `hkGlowStrong`,
  ширина 480. Поле поиска сверху («Search chats…», без отдельной
  подложки) + список отфильтрованных чатов ниже (пусто — «No chats
  found»), строка та же, что в сайдбаре, выбранная — подложка
  `hkAccentDim`. ↑↓ — навигация, Enter — открыть и закрыть палитру,
  Esc — закрыть без выбора, клик по строке — то же самое. Тени не
  добавляли — вариант без тени всех устроил.

### Единый хедер контент-панели (`ContentHeaderView`)

Один компонент на три состояния — обычный чат, чат внутри проекта,
страница проекта — заменяет прежний двухэтажный заголовок+крошку.

- Высота ровно 52pt, фон `hkPanel`, снизу hairline-разделитель 1px —
  тот же `Color.white.opacity(0.06)`, что у всех разделителей сайдбара
  (переиспользование существующего значения, не новый цвет).
  Горизонтальные отступы контента — 20pt, вертикальный центр.
- **Чат без проекта** — название (`hkBodyEm` = 13pt semibold, `hkInk`),
  тултип с полным текстом при усечении.
- **Чат внутри проекта** — иконка `folder` (13pt, `hkMuted`) → 6pt →
  имя проекта (`hkBody.weight(.medium)` = 13pt medium, `hkMuted`) → 8pt
  → «/» (`hkBody`, `hkNeutral`) → 8pt → название чата (`hkBodyEm`,
  `hkInk`). Иконка+имя проекта — одна кликабельная зона
  (`ProjectBreadcrumbButton`): без подложки и подчёркивания, при
  наведении и текст, и иконка становятся `hkAccent2`, курсор —
  pointing hand (через `NSCursor`, SwiftUI на macOS не даёт этого
  декларативно). Клик открывает страницу проекта.
  - Усечение при нехватке ширины: имя проекта усекается первым (до
    минимума 60pt, `.frame(minWidth: 60)`), название чата — последним
    (`.layoutPriority(1)`). У обоих — тултип с полным текстом.
- **Страница проекта** — хедер пустой (только фон и разделитель), имя
  проекта живёт в теле страницы, не дублируется в хедере.
- **Бейдж субагентов** (см. §7) — правый край, всегда в естественном
  размере; при нехватке места хедер жертвует названием/крошкой первыми.

### Проекты (`Project`)

Группа чатов с общими инструкциями и общей памятью (аналог проектов в
Claude) — отдельная SwiftData-сущность, Sessions API-only.

- **Секция в сайдбаре** — между «Pinned» и «Chats»: Pinned → Projects →
  Chats. Видна всегда (даже без единого проекта, как «Chats»), со своей
  кнопкой «+» — создаёт проект сразу («New project»), без модального
  окна, и сразу открывает его страницу.
- **Строка проекта** (`ProjectRow`) — та же анатомия, что у чата
  (`ConversationRowContent`), но с иконкой папки слева и количеством
  чатов вместо времени последней активности («3 chats»). Наведение → «…»
  (`ConversationMenuButton`, только Delete — переименование только на
  странице проекта, см. ниже).
- **Чаты проекта не видны в секции «Chats»** — только на странице своего
  проекта (в Cmd+K-палитре — видны, как и все чаты, фильтра там нет).
  Создать чат внутри проекта можно только с его страницы — существующие
  чаты в проект не переносятся (осознанное решение: у них другая
  память).
- **Страница проекта** (`ProjectView`, замещает панель чата целиком,
  хедер — пустой вариант `ContentHeaderView` выше) — колонка
  `min(680, ширина панели − 48)` по центру (минимум 24pt по бокам от
  края панели), отступ 32pt от хедера. Сверху вниз:
  - **Имя** — 22pt semibold, `hkInk`, без рамки в покое. Локальный
    черновик, не живая правка `project.name`: сохраняется по потере
    фокуса и по Enter (Enter снимает фокус), пустое значение при потере
    фокуса откатывается на прежнее. При наведении/фокусе — фон
    `hkSurface`, скругление 6pt, поле «выезжает» на 8pt без сдвига
    текста (компенсирующий отрицательный `padding`); при наведении, но
    не в фокусе — иконка pencil (12pt, `hkNeutral`) справа через 8pt, в
    фокусе скрыта.
  - 24pt → метка «INSTRUCTIONS» (11pt semibold, `hkNeutral`, tracking
    0.5) → 8pt → поле инструкций: `GrowingTextEditor` (см. §6) в
    режиме без автофокуса и без Enter-to-send, фон `hkSurface`,
    скругление 8pt, padding 12pt, высота 96–240pt (дальше — внутренний
    скролл), плейсхолдер «Add instructions for all chats in this
    project…» (`hkNeutral`). В фокусе — обводка `hkAccentDim` 1px
    (ближайший к «акцентному glow» существующий полупрозрачный ржавый
    токен — своего accent-варианта `hkGlowStrong` в палитре нет).
    Автосохранение — дебаунс 500мс после остановки ввода (отменяемая
    `Task`, без Combine). → 6pt → подсказка «Sent with every message in
    this project — keep it short.» (`hkCaption`, `hkNeutral`).
  - 32pt → метка «CHATS» (тот же стиль) + счётчик числом через 6pt,
    справа в той же строке — кнопка «New chat» (12pt medium,
    `hkAccent2`, иконка plus 11pt слева через 4pt, без фона в покое,
    hover — `hkSurface`, скругление 6pt, padding 4×8pt) → 8pt → список
    чатов проекта — та же строка/меню, что у обычного чата (общий
    `ChatSidebarViewModel`/`RenameChatSheet`/`ConversationMenuButton`),
    но БЕЗ Pin: чат проекта закрепить нельзя, иначе неясно, должен ли
    он тогда всплывать и в общей секции «Pinned». Пусто — «No chats
    yet. Start one to work in this project.» (`hkBody`, `hkNeutral`, по
    центру колонки, отступ 24pt сверху).
- **Создание проекта** — «+» в сайдбаре открывает сразу страницу
  проекта (не чат), имя по умолчанию «New project», поле имени сразу в
  фокусе с выделенным текстом — только для только что созданного
  проекта (флаг живёт в `ContentView`, гасится после первого появления
  страницы; открытие уже существующего проекта фокус не трогает).
- **Удаление проекта** — только из «…» в строке сайдбара. Алерт «Delete
  project?» с текстом «This will permanently delete this project and
  its N chats.» (N — реальное число чатов на момент клика). При
  подтверждении сначала удаляется серверная сессия каждого чата (`DELETE
  /api/sessions/{id}`), затем сам проект (локальный каскад удаляет его
  чаты). Если хоть одно серверное удаление не удалось — **кроме 404**
  (сессия и так уже удалена, пропускается и не считается ошибкой) —
  процесс останавливается, проект и чаты НЕ удаляются, показывается
  отдельный алерт с ошибкой; повторное «Delete» продолжает с того места,
  где остановилось (`ProjectSidebarViewModel.confirmDelete`) — так
  неудачное частичное удаление не оставляет проект в подвешенном
  состоянии и не плодит дублирующиеся запросы на уже удалённые сессии.
- **Инструкции и общая память в запросах** — единственный транспорт для
  чатов проекта, `SessionsConversationService`, на каждый `POST
  /api/sessions/{id}/chat/stream` заново читает `chat.project?.instructions`
  и `chat.project?.sessionKey` (не кэширует их при создании сервиса —
  правка инструкций посреди разговора действует уже на следующее
  сообщение, ожидаемое поведение). Пустые/только-пробельные инструкции
  не отправляются вовсе — поле `instructions` опускается из тела запроса
  целиком, а не шлётся пустой строкой. `sessionKey` идёт заголовком
  `X-Hermes-Session-Key`, только если чат в проекте; для чатов вне
  проекта ни поле, ни заголовок не отправляются — там ничего не
  изменилось.
  - ⚠️ **`X-Hermes-Session-Key` НЕ даёт изоляции памяти между
    проектами** — проверено экспериментом 2026-07-11 (см.
    docs/HANDOFF.md): сервер заголовок принимает и эхо-возвращает, но
    память Ржавчика (файловая, через git-синк agents-hub) в любом случае
    одна на все чаты, в проектах и вне них. Заголовок посылается на
    будущее, изоляция памяти проектам не обещана.
  - Поле `instructions`, наоборот, подтверждено рабочим: сессия
    получает `has_system_prompt: true`, агент следует инструкции
    (надёжно — на новой сессии; в сессии с уже идущей историей может не
    сработать сразу, обычное поведение модели).

### Абстракция беседы (`ConversationService`)

`ChatView`/`ChatViewModel` не знают, каким транспортом обслуживается чат
на экране — оба скрыты за протоколом `ConversationService`
(`Core/Conversation/`): `RunsConversationService` (для закреплённых
чатов на старом Runs API, поведение не менялось) и
`SessionsConversationService` (Sessions API). Маппинг событий стриминга
обоих API в общий `RunEvent`/`RunEventType` живёт по одному разу на
диалект: `SSEClient` (Runs: `message.delta`/`tool.call`/`tool.result`)
и `SessionSSEClient` (Sessions: `assistant.delta`/`tool.started`/
`tool.completed`) — низкоуровневое чтение SSE-потока у них
продублировано намеренно (не выносили в общий код), чтобы не рисковать
уже рабочим стримингом закреплённых чатов при доработке остальных.

⚠️ У Sessions API нет отдельного эндпоинта «остановить генерацию» —
кнопка Stop для чата на Sessions API обрывает локальное чтение потока
(`SessionsConversationService.stop()`), а не отменяет генерацию на
сервере, как это делает Stop для закреплённого чата на Runs API (`POST
/v1/runs/{id}/stop`).
